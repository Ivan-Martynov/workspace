Basics

Initial Structure

The <!DOCTYPE html> tag

A void, non-closing tag, and, guess what? It tells the browser you are using... html! You didn't expect that coming, right? Alright, to be more specific, it is HTML5, that standard for 2026. Who knows, maybe there will be HTML6 and things change a lot. But so far the HTML5 along with CSS3 are "living standards", and we all hope it is gonna stay that way, those version evolving without breaking changes. However, never say never.

The <html> tag

Besides being the tag to embrace all the staff, it often has attribute lang with the value to specify page's language.

Head

You know, keep metadata here, link external resource, give the title to the website – all those familiar elements to place under the hood.

Meta

It is not unusual to have quite a few meta tags. And they are all non-closing, void tags. We keep web page encoding information here, usually unicode as of 2026: <meta charset="UTF-8" />

Another common specification: <meta name="viewport" content="width=device-width, initial-scale=1.0" /> – follow the device width and zoom level at one, that is, no zooming in or out.

Body

Naturally, body under the head. Here we place the main content – no surprises.

Headings

So you know how to start your html, right? Doctype, head, body and yada yada. Now headings. The biggest is <h1>, the smallest is <h6>. Don't skip when going to the next heading in the hierarchy of your web page. Okay? After <h1> don't put <h3> or <h5>, always <h2>. Good! Done.

Paragraphs

That's a paragraph made with the <p>span tag. A block element in the html page. Everybody knows that, done.

Images

That's one without a closing tag, often closed with a slash, kind of self-closing element. Sometimes seen without a slash. Up to you. Or the guidelines of your superiors.

Need a src attribute, otherwise what's the point of having an <img> element in the first place? Provide a valid path to the image in the src attribute, otherwise no image!

Add an alt attribute. It will work without one, but it's appreciated to have a description of the image.

Strict tools will require the width attribute for the image element. So, you know, it's good to have it.

One important element for adding images is svg, which stands for Scalable Vector Graphics. As the name implies, this image can be resized without losing quality whatsoever. Usually this format is recommended for icons.

Links

The <a> elements

The well known <a> element. Attributes like target with value of _blank are quite familiar, right? And of course, you never forget to specify the source with the href attribute, do you?

Don't forget the order when styling states: link (the user hasn't yet interacted with your link), visited (the user has already visited the place where the link sends to), hover (give some attention to the link when a cursor over), focus (when the link is focused, for example, by tabbing to it) and then active (when the link is being clicked). The order is important, so remember it: LVHFA. If you mess up the order, some styles won't work.

The <link> elements

Another type of link – linking to external resource. Actually, these guys belong to the head, we don't put them in the body. Often used to attach a stylesheet file to the html file. I often forget what attributes are proper and how fill their values. Often you need a href attribute with the path to that resource. For a stylesheet you need the rel attribute with value stylesheet

Another example is connecting to a codebase,for example, to use a Google Font. In this case you might see the rel attribute with value preconnect and attribute crossorigin with no value.

One more example is linking an icon: rel="icon" with the path to an icon file.

Containers

The <div> tag

Group the elements – easy! Style them.

The <section> tag

A more semantic block to logically structure the content.

Attributes

The <id> attribute

Unique thing, can't use for other elements. The id attribute cannot contain spaces, can only contain letters, digits, underscores and, ta-da, dashes, which is a tad more what we are limited to when declaring a variable in most programming languages. Referenced with the hash symbol in css.

The <class> attribute

Not a unique thing, can use for other elements. Also can use several classes for one element. The class attribute can contain spaces, which basically means multiple classes separated by spaces. Referenced with the dot symbol in css.

You almost always want to use classes as they help apply same styles to different elements. Using id is not that common -- we need to really-really-really need the element to be unique.

Entities

Entities are used to display characters, which are reserved for html code. For example, using entity &lt; (less than) to display an opening tag. Combining those we could, for example, print tag for an image: &lt;img /&gt; => <img /> The entities can be represented by named references as we saw the use of &lt; and &gt;, by decimal or hexadecimal references: &#42; => * and &#x03F4; = ϴ

Script Element

Most likely, you know what it does. What's the web page nowadays without interactivity provided by javascript. So we embed the executable code providing the path to the javascript file: script element with attribute src for the path.

Search Enginge Optimization (aka SEO)

Enabling search engines to find your web page and understand what it is about. Done by the meta tag, with name attribute set to description and content attribute with the description of the page. Also, the title of the page is important for SEO, so don't forget to set it in the head.
Open Graph protocol is also important for SEO, as it allows to specify how the page should be displayed when shared on social media. It is done with the meta tag with property attribute set to og:title, og:description, og:image and so on, and content attribute with the corresponding value. Most important properties are url, image, type and title. The type can be website, article, video and so on. The image should be of good quality, as it will be displayed when the page is shared on social media. The title and description should be catchy and informative, as they will be displayed in search results and when the page is shared on social media.

Audio and Video

The <audio> element allows audio on the web page with the src attribute defining the path to the audio file. But in order to see and control the element, you better add the controls attribute. Another attributes which are good to have are loop and muted.

The nice thing about the audio element is that we can provide different sources between the opening and closing tag of the audio element. And the sources are added with the help of source elements, each containing the src attribute for the path and type attribute (e.g., audio/ogg, audio/wav etc.). The idea is that the browser will open the first source it can play thus the other sources serving as fallbacks.

The <video> element has same attributes as the audio elemnt, plus having autoplay, width and poster attributes. The poster specifies the image to show while the video is loading. The video element can also have fallback sources.

Embedding Content

The iframe element allows embedding the content of another web site. Often used to show maps. Naturally, contains the src attribute. You can also set the size of the frame via width and height attributes. The title attribute is self explanatory. The allow attribute defines the so called allowlist, telling what iframe can do.
